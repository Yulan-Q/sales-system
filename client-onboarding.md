# 客户接入流程 (Client Onboarding)

---

## 📋 流程概述

**目标**: 让客户在 3 天内顺利使用系统，产生第一条线索，发送第一封邮件

**时间线**:
```
Day 0: 签约付款
Day 1: 需求确认 + API 配置
Day 2: 系统部署 + 测试
Day 3: 培训上线 + 开始使用
```

---

## 🎯 Day 1: 需求确认 + API 配置

### 上午：需求确认会议（30 分钟）

**会议议程**:

| 时间 | 内容 | 负责人 |
|------|------|--------|
| 5 分钟 | 双方介绍 + 会议目标 | 双方 |
| 10 分钟 | 了解客户业务 | 我们提问 |
| 10 分钟 | 确认产品/市场/目标 | 我们提问 |
| 5 分钟 | 确认对接人 + 沟通方式 | 双方 |

**需求调研问题**:

#### 业务信息
- "贵公司主要出口什么产品？"
- "年出口额大概多少？"
- "外贸团队有几个人？"
- "目前主要用什么方式获客？"

#### 目标市场
- "主要做哪些国家/地区？"
- "哪个市场效果最好？"
- "有没有想开发的新市场？"

#### 客户画像
- "您的典型客户是什么类型？" (批发商/零售商/品牌商)
- "客户一般从哪里采购？"
- "采购决策流程是怎样的？"

#### 产品优势
- "相比竞争对手，您的核心优势是什么？"
- "有没有认证/专利/独特技术？"
- "MOQ 和交期是多少？"

#### 现有资源
- "有没有现成的客户列表？"
- "有没有官网/阿里店/社媒账号？"
- "有没有产品目录/报价单？"

**输出**: 《客户需求确认表》

---

### 下午：API 配置

#### 1. 帮客户注册 Apify

**步骤**:
1. 访问 https://console.apify.com/
2. 点击"Sign Up"
3. 用客户邮箱注册
4. 验证邮箱
5. 进入 Account → Integrations
6. 复制 API Token

**费用**: 
- 免费计划：$5/月，100 次 Actor 运行
- 推荐计划：$49/月，1000 次运行

**话术**:
> "王总，Apify 是我们的数据源提供商，需要您注册一个账号。费用是 $49/月，直接付给 Apify。我这边可以帮您注册，您看是用公司邮箱还是您的个人邮箱？"

---

#### 2. 帮客户注册 Brevo

**步骤**:
1. 访问 https://www.brevo.com/
2. 点击"Sign Up Free"
3. 填写公司信息
4. 验证邮箱
5. 验证域名（重要！）
6. 进入 Settings → API Keys
7. 创建新 Key

**域名验证步骤**:
1. 登录域名管理后台（阿里云/腾讯云等）
2. 添加 TXT 记录
3. 等待 DNS 生效（5-10 分钟）
4. 在 Brevo 点击验证

**费用**:
- 免费计划：每天 300 封
- 推荐计划：$25/月，5000 封/月

**话术**:
> "Brevo 是邮件发送服务商，免费计划每天可以发 300 封，对初创企业够用了。后期如果量大，再升级付费计划。"

## 🚀 Day 2: 系统部署 + 测试

### 上午：系统部署

#### 1. 配置 Agent

```bash
# 为客户创建独立配置
cp -r ~/templates/cross-border-agent ~/clients/[客户公司名]
cd ~/clients/[客户公司名]

# 修改配置文件
vi config/agent-config.json

# 填入客户专属信息
{
  "company_name": "客户公司英文名",
  "products": ["产品 1", "产品 2"],
  "advantages": ["优势 1", "优势 2"],
  "target_countries": ["美国", "德国"],
  "api_keys": {
    "apify_token": "客户 Apify Token",
    "brevo_key": "客户 Brevo Key"
  }
}
```

#### 2. 配置邮件模板

根据客户产品和优势，定制开发信模板：

```markdown
【初次开发信模板】

Subject: Partnership Opportunity with [客户公司名]

Dear [联系人姓名],

I noticed that [客户公司名] specializes in [客户产品], and I'm impressed by 
your [具体亮点，如 product range/market presence].

We are [我方公司名], a professional manufacturer of [我方产品] with [X] years 
of experience. Our main advantages include:

• [优势 1 - 如 Competitive pricing with high quality]
• [优势 2 - 如 Fast delivery within 15 days]
• [优势 3 - 如 CE, RoHS, FCC certified]

We've been helping companies like yours [价值主张，如 reduce costs by 20% 
/ expand product range].

Would you be interested in seeing our latest catalog? I'd be happy to send 
you some free samples for evaluation.

Looking forward to your reply.

Best regards,
[姓名]
[职位]
[公司名]
[网站]
[WhatsApp/WeChat]
```

#### 3. 配置线索挖掘参数

```json
{
  "keywords": ["客户产品关键词英文"],
  "target_countries": ["目标国家英文"],
  "exclude_countries": ["不做国家"],
  "min_results": 200,
  "output_format": "csv"
}
```

### 下午：系统测试

#### 测试 1: 线索挖掘

```bash

node scripts/run-lead-search.js \
  --keyword "[产品关键词]" \
  --country "[目标国家]" \
```

**验收标准**:

- ✅ 每条包含公司名/网站/邮箱
- ✅ 邮箱格式正确
- ✅ 数据可导出

---

#### 测试 2: 开发信生成

```bash
# 执行测试

  --lead "data/leads/[日期]_leads.csv" \
  --template "templates/email-initial.md" \
  --output "data/emails/"

# 检查结果
cat data/emails/[客户名]_email.md
```

**验收标准**:
- ✅ 邮件包含客户公司名
- ✅ 提到客户产品/业务
- ✅ 突出我方优势
- ✅ 有明确行动号召
- ✅ 语法正确，无错别字

## 📚 Day 3: 培训上线

### 上午：系统培训（1 小时）

**培训议程**:

| 时间 | 内容 | 方式 |
|------|------|------|
| 10 分钟 | 系统概述 | 讲解 |
| 15 分钟 | 线索挖掘演示 | 实操 |
| 15 分钟 | 邮件发送演示 | 实操 |
| 10 分钟 | CRM 使用演示 | 实操 |
| 10 分钟 | 常见问题解答 | 问答 |

**培训材料**:
- 演示账号
- 操作录屏
- 使用手册
- 常见问题文档

**培训话术**:

**开场**:
> "大家好，我是 [名字]，今天由我来给大家培训系统的使用。整个培训大概 1 小时，我会边演示边讲解，大家有任何问题随时打断我。培训结束后，我会把操作手册和录屏发给大家。"

**线索挖掘演示**:
> "好，现在我们来看第一个功能 - 线索挖掘。大家看到这个界面，这里有三个输入框：
> 1. 产品关键词 - 用英文，比如'LED light'
> 2. 目标国家 - 可以选一个或多个
> 3. 线索数量 - 建议每次 100-500 条
> 
> 填好之后，点这个'开始挖掘'按钮... 大家看，系统开始工作了，大概需要 10-30 秒... 好了，找到了 87 条线索。
> 
> 大家看这些字段：公司名、网站、邮箱、电话、地址，都很完整。我们可以直接导出 Excel，也可以直接在系统里操作。
> 
> 来，我导出给大家看看...（导出 Excel）好，大家看这个文件，数据都在里面了。销售同事拿到这个，就可以开始联系了。
> 
> 有什么问题吗？"

**邮件发送演示**:
> "接下来我们看邮件发送功能。找到刚才的线索，点这个'生成开发信'按钮...
> 
> 大家看，系统会自动分析这个客户的网站，然后生成一封个性化的邮件。注意看这里，邮件里提到了客户的公司名和产品，这就是个性化的体现。
> 
> 如果觉得没问题，点'发送'，邮件就发出去了。系统会自动追踪客户有没有打开、有没有点击。
> 
> 我们来看一下追踪数据... 好，这封邮件客户已经打开了，还点击了网站链接。这种就是高意向客户，系统会自动标记出来。
> 
> 大家有什么问题吗？"

**CRM 演示**:
> "最后我们看一下客户管理。所有联系过的客户都会在这里显示。大家看这个销售漏斗：
> - 新增线索：还没联系的
> - 已联系：发了开发信的
> - 已回复：客户回复了的
> - 报价中：正在谈价格的
> - 样品：寄了样品的
> - 成交：已经下单的
> 
> 每个客户的详细记录点进去都能看到，包括邮件历史、联系记录、备注等等。
> 
> 老板还可以在这里看到团队业绩，每个人联系了多少客户、有多少回复、成交了多少单，都清清楚楚。
> 
> 好，系统功能就演示到这里。大家有什么问题吗？"
