### 扩展点 4: 评分引擎 (ScoringEngine)

**当前实现**:
- ⏳ 基础规则评分（权重配置）

**未来扩展** (插件化):
```javascript
// services/scoring/scoringEngine.js
class ScoringEngine {
  constructor() {
    this.scorers = [
      new DomainScorer(),      // 邮箱域名质量
      new CompanyScorer(),     // 公司规模
      new IndustryScorer(),    // 行业匹配
      new LocationScorer(),    // 地理位置
      // 未来可扩展
      new SocialScorer(),      // 社交媒体活跃度
      new TechnographicsScorer(), // 技术栈分析
      new IntentScorer()       // 购买意向信号
    ];
  }
  
  async score(lead) {
    let totalScore = 0;
    for (const scorer of this.scorers) {
      totalScore += await scorer.score(lead);
    }
    return Math.min(totalScore, 100);
  }
}
```

## 📊 数据库扩展设计

### 核心表结构
```sql
-- 线索表（核心）
CREATE TABLE leads (
  id INTEGER PRIMARY KEY,
  company_name TEXT,
  email TEXT,
  phone TEXT,
  country TEXT,
  industry TEXT,
  status TEXT, -- new/contacted/interested/qualified/invalid
  score INTEGER DEFAULT 0,  -- AI 评分 0-100
  source TEXT,  -- apify/manual/import
  assigned_to INTEGER,  -- 分配给哪个销售
  created_at TIMESTAMP,
  updated_at TIMESTAMP,
  
  -- 公海池相关
  is_in_public_pool BOOLEAN DEFAULT FALSE,
  auto_recycle_at TIMESTAMP,
  
  -- GDPR 合规（预留）
  data_source TEXT,
  gdpr_consent BOOLEAN DEFAULT FALSE,
  unsubscribed BOOLEAN DEFAULT FALSE
);

-- 邮件表
CREATE TABLE emails (
  id INTEGER PRIMARY KEY,
  lead_id INTEGER,
  campaign_id INTEGER,
  subject TEXT,
  body TEXT,
  status TEXT, -- sent/opened/clicked/replied/bounced
  sent_at TIMESTAMP,
  opened_at TIMESTAMP,
  clicked_at TIMESTAMP,
  
  -- 追踪相关
  tracking_pixel_id TEXT,
  links_clicked TEXT[]
);

-- 营销活动表
CREATE TABLE campaigns (
  id INTEGER PRIMARY KEY,
  name TEXT,
  status TEXT, -- active/paused/completed
  total_leads INTEGER,
  sent_count INTEGER,
  open_rate REAL,
  reply_rate REAL,
  created_at TIMESTAMP
);

-- AI 生成记录表（新增）
CREATE TABLE ai_generations (
  id INTEGER PRIMARY KEY,
  lead_id INTEGER,
  template_type TEXT,
  generated_content TEXT,
  model_used TEXT, -- qwen/gpt/etc
  tokens_used INTEGER,
  created_at TIMESTAMP
);

-- 评分记录表（新增）
CREATE TABLE lead_scores (
  id INTEGER PRIMARY KEY,
  lead_id INTEGER,
  score INTEGER,
  score_breakdown TEXT, -- JSON: {domain: 20, company: 30, ...}
  scored_at TIMESTAMP
);
```

## 🎯 核心原则

1. **快速迭代**: 先实现核心功能，再逐步完善
2. **模块化**: 每个功能独立，方便替换/升级
3. **可扩展**: 预留接口，新增功能不破坏现有代码
4. **可测试**: 每个服务可独立测试
5. **可观测**: 关键操作记录日志，方便调试
